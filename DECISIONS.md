# Solo RPG Companion App — Decisions & Direction

*Last updated: 2026-07-17*

## Product identity

A personal desktop companion app for playing solo tabletop RPGs. The guiding stance: **the app is the table, not the notebook**.

- **No journal feature — deliberately.** Paper journaling is part of the play experience and the app should encourage it, not replace it. The app owns the digital clutter (book, dice, cards, music, state) so the notebook stays analog.
- Local-first: no accounts, no cloud, data in plain local files.
- Built by one developer, for their own play (current game: *Artefact* by Mousehole Press — a solo journaling RPG — plus its Playkit, expansions, and soundtrack).

## Layout (decided)

- **PDF gamebook takes the entire screen.** The viewer is the backbone feature; everything else stays out of its way.
- **Bookmark tabs along the right edge** of the screen.
- **Tools live in compact chrome** — icons, tabs, or a bottom drawer (final form not yet chosen). Design guidance adopted: tools open *over* the page and dismiss instantly; rolling a die must never navigate away from the book. Semantic edges: **right edge = places in books, bottom edge = tools on the table.**
- **Multiple reference PDFs** are first-class (character sheets, playkits, expansions) — needs a way to switch between open documents, not just pages within one.

## Features

### Decided
| Feature | Notes |
|---|---|
| PDF viewer + bookmarks | Full-screen, right-edge bookmark tabs, multiple documents |
| Dice roller | Functions like Google's search dice widget (see spec below) |
| Playing-card deck | 52 cards + optional jokers (toggle applies on next shuffle). **Session-only, no persistence** — decided 2026-07-17, overriding the earlier "decks with memory" recommendation; revisit if depletion-across-sessions starts to matter in play |
| Tarot deck | Full 78 cards, reversals on by default (50/50, toggle applies on next shuffle). Session-only like the playing deck — decided 2026-07-17 |
| Coin flipper | Standalone tool (like Google's coin flip), not part of the dice tray — decided 2026-07-17 |
| Audio player | Folder-based soundtrack playback (see spec below) |

### Dice roller spec (modeled on Google's dice widget)
- Palette of die buttons (d4, d6, d8, d10, d12, d20); each tap adds a die to the tray → mixed pools like 2d6 + 1d20, formula displayed next to the total.
- Dice render as flat 2D polygons (inline SVG) for v1. **Planned upgrade: replace with 3D models later** — the `@3d-dice/dice-box` library (WebGL physics dice) is the identified path.
- Roll animation: each die wobbles (~600ms) while its number flickers (~55ms interval), staggered ~90ms die-to-die; total counts up after the last die settles.
- Tap a die in the tray to remove it; Clear wipes the pool.
- Working demo built: `https://claude.ai/code/artifact/b12116fe-c39d-412c-81f8-a1fefc2b256b` (source also saved as `dice-roller-demo.html` in the session scratchpad — the CSS keyframes and SVG shapes port directly to a Svelte component).

### Audio player spec
- Point at a folder (e.g. the existing Artefact soundtrack folder) and treat it as the playlist — no library, no importing, no metadata database.
- Seamless looping and 2–3 second crossfade between tracks (hard cuts break the mood).
- Mini-bar (track name / pause / skip) reachable without opening the full drawer.
- Audio state persists with the session — resume restores the playlist and track.
- Later upgrades: bookmark-attached tracks (flip to a chapter, get its theme offered); optional card-flip/dice sound effects as a setting, never a default.

### Recommended, not yet committed
- **Roll & draw history** — an ephemeral session ledger of every roll/draw with timestamps. Not a journal: it's a record of the tools, there to glance at while writing the story on paper. Strongest idea from the ideation pass.
- **Decks with memory** — shuffle order, drawn cards, and discards persist across app launches; reshuffle only on deliberate action. A correctness requirement for depletion-based card-prompt games. *Decided against for now (2026-07-17): the playing-card deck shipped session-only. Revisit for the tarot deck or if cross-session depletion matters in play.*
- **Resume where I left off** — reopening restores last page, open documents, deck state, and audio.
- **Pinned region clips** — rubber-band a rules table or stat block into a small overlay for "this table AND my current page" moments.
- **Custom random tables** — plain-text table format rollable from the same drawer as the dice; how the app supports future games.

### Explicitly rejected
- **In-app journal / notes** — paper journaling is the point.

## Architecture principles (adopted)

1. **Result first, animation second.** Every roll/draw result is generated by code (crypto RNG) *before* any animation plays; the animation is theater that lands on it. Enables the history ledger, keeps results trustworthy, and lets animation tiers be swapped (2D → 3D dice) without touching game logic.
2. **Persist everything that is table state.** Page position, open documents, deck order, audio track — sitting back down should be instant.
3. **Plain local files** (JSON) for state — human-readable, easy to back up. SQLite only if full-text PDF search is added later.
4. **Respect reduced motion**, and offer a "skip the ceremony" instant-result toggle.

## Tech stack (decided)

| Layer | Choice | Role |
|---|---|---|
| App shell | **Tauri 2** | Native window, file access (PDFs, soundtrack folder), small binary |
| UI framework | **Svelte 5 + TypeScript** | Components: drawer, bookmark tabs, decks, dice tray |
| PDF rendering | **PDF.js** | Full-screen gamebook view, page render, region render for clips |
| Audio | **Web Audio API** | Loop + crossfade playback |
| Storage | **JSON files** (via Tauri) | Bookmarks, deck state, session state |
| Animations v1 | **CSS transforms/keyframes + inline SVG** | Card flips (rotateY + backface-visibility), Svelte fly/flip transitions, dice tumble |
| Animations v2 | **@3d-dice/dice-box** | Physics-based 3D polyhedral dice on a transparent canvas |

Alternatives considered and set aside: Electron (too heavy), SwiftUI/PDFKit (best native PDF and an iPad path, but steeper ramp and Apple-only), pure web/PWA (local-file friction), Raylib (no document UI).

## First milestone

Prove the PDF experience before anything else: a bare Tauri + PDF.js prototype opening the actual Artefact Main Book — smooth scrolling, zooming, and page-caching on a big art-heavy PDF. It's the riskiest piece; every other tool is a weekend-sized feature once the backbone works.

## Project status (2026-07-17)

Scaffolded at `solo-rpg-companion/` (Tauri 2 + SvelteKit + TypeScript, via create-tauri-app):

- `src/lib/PdfViewer.svelte` — the milestone-one viewer: continuous scroll, lazy render via IntersectionObserver (~1.5 screens ahead, canvases freed when scrolled away), fit-width + zoom (50–400%), retina-aware rendering, current-page tracking.
- `src/routes/+page.svelte` — app shell: open-PDF dialog, floating low-opacity chrome (top-left: open/title; bottom-center: zoom + page count) that fades in on hover — first pass at the "chrome stays out of the book's way" principle.
- `src/lib/DiceTray.svelte` — the dice roller, shipped per the spec above: d4–d20 palette, mixed pools with formula + total, 2D SVG dice, staggered tumble/flicker/count-up animation, crypto-RNG results generated before the animation plays, reduced-motion = instant results. Opens from a "Dice" button in the bottom chrome as an overlay above it (tools-over-the-page pattern); Escape or the button dismisses it; the pool persists while closed. First tool built on the bottom-edge drawer concept.
- `src/lib/CoinFlip.svelte` — standalone coin flipper: one big SVG coin, tap it (or Flip) to flip, rotateY spin + H/T flicker landing on a crypto-RNG result, reduced-motion = instant. Opens from a "Coin" button in the bottom chrome; opening it closes the dice tray and vice versa (both tools share the spot above the chrome).
- `src/lib/CardDeck.svelte` — the playing-card deck: crypto Fisher–Yates shuffle (whole order decided up front, result-first), tap the pile to draw with a rotateY flip-in, corner-index + center-pip card faces, remaining/drawn counts, deliberate Shuffle button, jokers toggle (applies on next shuffle). Session-only state; persists while the tray is closed but not across launches. Chrome refactored to a single `activeTool` switch — Dice/Coin/Cards buttons, one tool open at a time. Every card drawn since the last shuffle stays visible in a scrollable spread next to the pile (rapid clicking supported — no animation lockout); only Shuffle clears it.
- `src/lib/TarotDeck.svelte` — the tarot deck: full 78 (22 majors with roman numerals + 56 minors), crypto Fisher–Yates shuffle with reversals baked into the shuffle itself (each card's orientation is decided when the deck is shuffled, like physical tarot — order and orientation both exist before any flip). Reversed cards render rotated 180° with a "— Reversed" label. Reversals toggle (on by default, applies on next shuffle), deliberate Shuffle, session-only state. Deep green card back with gold star, distinct from the playing deck's gold back. Opens from a "Tarot" button in the chrome. Drawn cards accumulate in the same scrollable spread as the playing deck until reshuffled.
- `pdfjs-assets` npm script rewritten in Node (was Unix `rm`/`cp`, which broke `npm run dev` on Windows).
- Rust side: `tauri-plugin-dialog` + `tauri-plugin-fs` registered; capability grants file reads under `$HOME` and `/Volumes`.
- Run with `npm run tauri dev` inside `solo-rpg-companion/`.
